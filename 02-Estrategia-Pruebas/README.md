ESTRATEGIA DE PRUEBAS - SAUCE DEMO

Objetivo: Definir la estrategia de pruebas, para validar el correcto funcionamiento de la aplicación Sauce Demo, asegurando que las funcionalidades de aplicativo cumplan con los requisitos esperados antes de su liberación.

Alcance de prueba

Las pruebas incluiran la validación funcional de las siguientes áreas:
- Autenticación
- Visualización del catálogo de productos
- Gestión del carrito de compras
- Proceso de checkout
- Finalización de la compra
  
Tipos de Prueba

Durante el proceso se ejecutaran los siguientes tipos de prueba:
- Pruebas funcionales
- Pruebas de regresión
- Smoke testing
- Pruebas negativas

Ambiente de Pruebas
- Aplicación Web
- Navegador Web: Chrome
- Ambiente público de pruebas}

Criterios de Entrada
- La aplicación debe estar disponible
- El ambiente debe ser accesible
- Las credenciales de prueba deben ser validas

Criterios de Salida
- Los casos de prueba criticos deben ejecutarse
- Los defectos criticos deben estar documentados
- Se debe contar con evidencia de las pruebas ejecutadas
